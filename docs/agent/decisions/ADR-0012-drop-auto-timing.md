# ADR-0012 — 타이밍 자동 정리 폐지 (ADR-0009 일부 supersede)

- Status: Accepted
- Date: 2026-07-17
- Area: editor / timing
- Supersedes: ADR-0009의 "자동 타이밍 정리(absorb → 발화스냅 + 초과분할 + 확장)" 부분.
  ADR-0009의 나머지(내용/타이밍 모드 분리, `review_flag`, 흘려듣기, 문제 큐)는 유효.
- Decision Owners: User (폐지 확정) + Agent (계측·대안 제시)

## Context

ADR-0009는 `✨ 타이밍 자동 정리`를 채택했다: 캐시된 단어 타임스탬프로 (1) 모든 셀을
발화에 스냅, (2) 예산 초과 셀을 가장 넓은 내부 pause에서 분할, (3) 너무 빠른 셀의
표시 시간을 늘림. 무-API·결정적이라 검수자에게 열려 있었다.

사용자 보고 (2026-07-17):
> "성과가 형편없다. 강연이라 **다른 사람이 말하던 부분까지 포함**하고, **제대로 수정한
> 자막도 이상하게 나누거나** 타이밍을 이상하게 잡는다."

**계측** (STT 블롭 보유 18개 영상 / 7,760셀에 `plan_track`을 읽기 전용 실행):

| 지표 | 값 |
|---|---|
| 건드리는 셀 | **5,138 / 7,760 = 66%** |
| 검수 완료 영상(p_9m8r1bZSM, 2,231셀) | **98%** — 사실상 전면 재작성 |
| 셀 텍스트 ↔ 구간 STT 단어 일치도 <0.6 | 1,384 (18%) — 그중 **544셀을 그대로 변경** |
| 일치도 중앙값 | 검수 완료 0.80 vs 미검수 1.00 |

구조적 원인 (`pipeline/retime.py`):
1. snap이 조건 없이 전 셀에 적용 → 이미 맞는 셀도 흔든다.
2. `plan_track`이 단어의 **텍스트를 버리고** `(start, end)`만 받아 분할 위치를
   **시간 비례**(`len(text) * ratio`)로 정한다. 사람이 텍스트를 고칠수록 이 비례식이
   깨져 엉뚱한 자리를 자른다 — **검수가 진행될수록 도구가 더 나빠진다**.
3. Whisper는 화자분리를 하지 않으므로 청중·질문자 발화가 구간에 들어오면 snap이
   거기 끌려가고 그 pause에서 분할된다.

## Decision

**폐지한다.** 개선(정렬 근거 + 임계 + 미리보기 모달)을 설계했으나(ACTIVE_PLAN),
사용자가 **"어차피 잘 쓰지 않는다"**며 폐지를 택했다.

- 실검수에서 `✨ 타이밍 자동 정리` 버튼 제거.
- `/api/jobs/{id}/auto-timing`은 **실영상에 대해 410**을 돌려준다 — 프론트에서 버튼만
  빼면 **캐시된 옛 탭**이 그대로 호출할 수 있고(실제 관측), 그러면 검수 완료 영상의
  98%가 재작성된다. 서버가 최종 방어선.
- 연습(practice) 클론에서는 **재렌더 전까지 허용** — 연습5 나레이션이 아직 이 버튼을
  지시한다. 샌드박스라 실데이터 영향 없음 (안심 확인·복구 채우기와 같은 처리).

**남는 타이밍 도구** (전부 사람이 겨냥해서 쓰는 것):
`Alt+[` 여기서 시작 · `Alt+]` 여기서 넘김 · `Alt+\` 발화 맞춤 · 타임라인 손잡이 드래그 ·
`✂ 무음 다듬기` · `다음 문제 →`(이동만, 수정 안 함).

## Rationale

- 도구의 가치는 "맞을 때 아끼는 시간" − "틀릴 때 되돌리는 시간". 66%를 건드리면서
  근거 없는 변경이 섞이면 **사람이 전수 검사**해야 하고, 그건 손으로 하는 것보다 느리다.
- 특히 **검수가 끝난 영상일수록 정확도가 떨어진다**. 가장 필요한 순간에 가장 나쁘다.
- 개선안(정렬)은 기술적으로 가능하나(단어 텍스트가 블롭에 있음) 미리보기 모달까지
  붙여야 안전하고, 실사용 빈도가 낮아 **투자 대비 가치가 낮다** — 사용자 판단.
- 없애면 잘못 건드릴 일 자체가 사라진다. 남는 도구는 전부 사람이 한 셀씩 겨냥하는 것.

## Consequences

- `pipeline/retime.py`와 `/auto-timing` 엔드포인트는 **연습 전용**으로만 살아 있다.
  연습5 재렌더 후 **완전 삭제 대상** (그때 `retime.py`도 함께).
- **연습5 재설계 필요** — 나레이션 3·12번 줄이 자동 정리를 지시한다. 재설계계획
  §6에 반영: 스텝 9개 → 8개, "자동 정리 먼저" 흐름이 "문제만 골라 손보기"로 바뀐다.
- ACTIVE_PLAN(타이밍 v2 "확실한 것만")은 **폐기** — 계측 결과는 이 ADR에 보존.
- 타이밍 품질은 이제 전적으로 STT 분절 + 사람 손에 달렸다. STT가 앞 침묵을 물고 끊는
  문제(4~19% 셀이 최대 3~4초)는 `✂ 무음 다듬기`와 `Alt+\`로 대응.
- **미해결로 남는 것**: `✂ 무음 다듬기`도 같은 snap 규칙을 쓴다(화자분리 없음, 정렬
  없음). 전 트랙 일괄이라 같은 계열의 위험이 있다 → 사용자 관측 필요, 문제되면 동일
  절차로 재검토.

## Revisit Trigger

- 워커에 diarization(pyannote 등)이 들어와 화자 구간을 알 수 있게 되면 — 그때는
  "다른 사람 발화 포함" 원인이 사라지므로 정렬 기반 자동 정리를 재검토할 수 있다.
- `✂ 무음 다듬기`에서 같은 불만이 관측되면.
