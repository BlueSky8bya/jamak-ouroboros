# ADR-0006: 언어별 독립 자막 트랙

Status: Accepted
Date: 2026-07-11
Decision Owners: User + Agent-assisted

## Context

지금 모델은 세그먼트(한국어)가 유일한 구조이고, 번역은 `Translation(segment_id, lang, text)`로 **한국어 세그먼트 구조·타이밍에 종속**된다. 그러나 언어마다 읽기 속도·어순이 달라(한국어 1줄 = 영어 2줄 등) 자막을 **언어별로 다르게 분할/병합/타이밍**해야 하는 것이 자막 업계 표준(Aegisub·Subtitle Edit·Netflix: 각 언어 = 독립 자막 파일). 또한 다중 사용자로 확장 시 "누구는 한국어 텍스트, 누구는 영어 타이밍"처럼 **(영상×언어×단계)로 작업을 세분화**해야 한다.

## Decision

**모든 언어를 1급 자막 트랙으로 승격**한다. 트랙 = `(job, lang)`에 속한 세그먼트 묶음.

- `Segment.lang` 추가(기존 행 = `"ko"`). 한국어 = 소스 트랙.
- `번역 만들기` = 한국어 트랙의 구조·타이밍을 복사 + 번역 텍스트를 채워 **새 lang 트랙을 시드**. 이후 그 트랙은 독립적으로 분할/병합/재타이밍 가능.
- 기존에 만든 에디터(분할·병합·WordMap·타이밍·미리보기·무음다듬기)는 **선택된 lang 트랙의 세그먼트**에 그대로 작동 → 전 언어 재사용.
- 트랙별 상태: 텍스트 검수(세그먼트 reviewed 파생) + 타이밍 검수(`Track.timing_done`). `Job.timing_done`(ko 전용)은 `Track(ko)`로 이관.
- 랜딩은 언어를 축으로: (영상×언어)별 단계 태그(텍스트 검수중/완료, 타이밍 작업중/완료, ...).

## Rationale

- 언어별 가독성·타이밍 독립 = 자막 품질의 필수 조건, 업계 표준.
- 에디터 전면 재사용 → 신규 편집 UI 거의 불필요.
- 다중 사용자 세분화(ADR 다음 단계)와 자연스럽게 맞물림.

## Consequences

- (+) 각 언어가 완결된 산출물. 전 언어에 정밀 타이밍 도구 적용.
- (-) **ko↔번역 1:1 매핑 상실**: 번역 트랙을 독립 편집(분할/병합)하면 한국어와 구조가 갈라짐. → 갈라지기 전에만 "한국어 바뀜→그 세그먼트 재번역"이 정밀. 이후엔 시간대 겹침 기반 "이 근처 원문 바뀜" 신호만(프로 툴과 동일).
- (-) 세그먼트 쿼리 전반이 lang-aware가 돼야 함(현재 `where(job_id==)` → `+ where(lang==)`). 마이그레이션·회귀 위험.
- (-) 우로보로스 학습(교정쌍·용어)은 ko 소스 전용 유지; 번역 학습은 별도(`translation_examples` 이미 존재).
- (-) 저장·복잡도 증가.

## Migration

- 추가형: `Segment.lang DEFAULT 'ko'`, `Track` 테이블 신설. 기존 `Translation` 데이터는 재번역 시점에 lang 세그먼트로 승격(또는 레거시 뷰 유지).
- 파괴 금지: `data/jamak.db` 기존 검수 데이터는 전부 ko 트랙으로 보존.

## Revisit Conditions

- 번역 트랙 독립 편집이 실제로 얼마나 갈라지는지 데이터로 확인 후 sync UX 조정.
- 다중 사용자/원격 검수(작업 분배·잠금) 설계 시.
