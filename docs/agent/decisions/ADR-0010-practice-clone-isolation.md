# ADR-0010: 튜토리얼 연습은 사용자별 Job 복제 샌드박스에서

- Status: Accepted (2026-07-14)
- Area: tutorial/data
- Related: docs/tutorial/PLAN.md v4 §4.3, CHG-20260714-010, ADR-0009

## Context

따라하기 코스용 전용 연습 영상(합성 TTS, `Job.practice`)에서 검수자들이
실습한다. 세그먼트는 전 사용자 공유(last-write-wins)라, A가 연습하며 고친
내용이 B·C의 연습 재료를 파괴한다. 사용자 요구 확정: "A가 아무리 연습·수정해도
B·C가 들어오면 **동시에, 병렬로, 각자 처음부터**". v3까지 검토했던
스냅샷+초기화(+동시 1명 운영)로는 이 요구를 충족하지 못한다 (Codex 감사
B1도 같은 지적).

또한 P5 리허설에서 대본에 심은 오인식 미끼가 whisper hotwords에 전부
패배(축지법·공중부양·뭉치 등 전부 정확 인식) — 확률적 재렌더 반복 대신
결정적 재료 확보가 필요했다.

## Decision

1. **기준(baseline) Job 불변**: 코스가 바인딩된 연습 Job은 리허설 통과 후
   아무도 직접 편집하지 않는다.
2. **사용자별 클론**: 연습 영상을 열면 서버가 기준 Job의 ko 세그먼트 전체 +
   SttBlob을 새 Job으로 깊은 복사. 식별은 브라우저별 UUID
   (`localStorage jamak.practiceKey`).
3. **합성 video_id**: 클론의 video_id = `"<base>~sha256(session_key)[:10]"`.
   기존 video_id 키 엔드포인트 전부가 무수정으로 클론에 동작하고, 플레이어만
   `~` 접미사를 떼고 실제 YouTube id를 로드한다. (접두 절단이 아닌 **해시** —
   절단은 UNIQUE video_id 충돌, E2E에서 실증.)
4. **초기화 = 재복제**: "처음부터 다시"는 클론 삭제 후 재복사. 세그먼트 id
   공간이 통째로 바뀌므로 늦게 도착하는 이전 저장 PUT은 자연 무효.
5. **코스당 활성 영상 1개**: `Job.practice_course` + 부분 UNIQUE 인덱스
   (`WHERE practice_course <> ''`). 코스 해제는 practice 플래그를 내리지
   않는다(합성 영상은 학습에서 영구 제외).
6. **결정적 결함 주입**: 코스 바인딩 시 서버가 기준 Job에 1회 주입
   (basic: 깨입·밥나무·축제법·공중부용 / fast: 몽치×2 / timing: 침묵 걸침).
   모든 클론이 동일 결함 상속 → 전 사용자 동일 드릴.
7. **위생**: 클론은 목록·튜토리얼 맵에서 숨김, practice=True라 모든 학습·평가
   경로에서 제외(CHG-20260714-005), 7일 유휴 시 워커가 삭제(clone_of NOT NULL
   AND practice=True 이중 조건 없이는 삭제 불가 — BR-DATA-001).

## Consequences

- (+) 병렬 격리가 구조적으로 보장 — 잠금·경합 처리·운영 규칙 불필요.
- (+) 엔드포인트 재사용: ps 파라미터 플럼빙(대안 설계) 대비 변경 면적 최소.
- (+) 연습 드릴이 결정적 — 리허설 재렌더 루프 제거.
- (−) 클론당 세그먼트 수백 행 + SttBlob 사본이 DB에 쌓임 → TTL 청소로 상쇄
  (연습 영상은 2분짜리라 행 수 작음).
- (−) `~` 접미사가 video_id 형식 가정에 스며듦 — 새 코드가 video_id로
  YouTube를 직접 부를 땐 반드시 접미사 제거 필요 (플레이어에 유틸 존재).

## Revisit Trigger

- 검수자 수가 커져 클론 볼륨이 문제될 때 (행 수 모니터링)
- 연습 영상에 번역 트랙이 필요해질 때 (현재는 ko 단일 트랙 계약으로 복제 거부)
- video_id 접미사 가정이 세 번째 이상 코드 경로에 등장할 때 (전용 해석 헬퍼로 승격)
