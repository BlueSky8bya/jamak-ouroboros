# PLAN-20260710-008 - Standalone Audience Response Filter

## Goal

Prevent standalone audience-response subtitles such as `네` from appearing in newly generated subtitle drafts.

## Risk Level

L2 - pipeline subtitle filtering behavior. No DB schema or seed-data mutation.

## Scope

- Add a deterministic noise-filter helper for short standalone audience responses.
- Apply it before crosscheck/DB segment creation so future jobs do not create these cells.
- Keep full sentences containing the same word intact.

## Result

Done. `src/jamak/pipeline/noise.py` filters conservative standalone replies (`네`, `네네`, `예`, `예예`, `넵`). `jamak run` applies it immediately after `split_segments`, so web-created jobs also benefit because the web job runner invokes the CLI pipeline.
